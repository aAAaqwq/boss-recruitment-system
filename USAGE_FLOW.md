# BOSS直聘招聘自动化 — 参考使用流程

## 典型招聘漏斗

```
  F5 打招呼          F6 获取简历         F7 批量回复
  ─────────         ─────────          ─────────
  推荐牛人页         聊天页              聊天页
  筛选 → 联系        已沟通的联系人       未读消息

  1000人浏览    →    100人打招呼    →    50人沟通    →    20人给简历
```

---

## 第一步：F5 筛选+打招呼（获取潜在候选人）

### 做什么

在 BOSS 直聘的"推荐牛人"页面，根据**学历、学校、工作年限**等条件筛选候选人，自动点击"打招呼"按钮发起沟通。

### 操作

```
① 在筛选项配置：学历 ≥ 本科、学校白名单、排除关键词
② 输入打招呼人数（默认 20，每日上限 80）
③ 点击 "👋 筛选+打招呼"
```

### 效果

程序在 iframe 内自动滚动候选人列表，对每个符合条件的卡片：
- 提取姓名、学历、学校、工作年限
- 匹配筛选条件
- 去重（同一个人不打两次）
- CDP 点击"打招呼"按钮
- 写入数据库 `candidates` 表（status=contacted）
- 遇到 BOSS 限制弹窗自动停止

### 结束后你得到什么

- 数据库里多了 N 个 `status=contacted` 的候选人记录
- BOSS 直聘聊天列表里多了 N 个"沟通中"的对话
- 这些候选人会收到你的招呼消息，部分人会回复

---

## 第二步：F6 批量获取简历（从已沟通的人里拿简历）

### 前提

只有**已经沟通过**的联系人才能请求/下载简历。所以必须先做完 F5（或手动打过招呼），等候选人回复后，再跑 F6。

### 做什么

在聊天页筛选"沟通中"的联系人，逐个点击，找到"附件简历"按钮，自动下载或请求简历。

### 操作

```
① 输入获取人数（默认 10）
② 点击 "📄 批量获取简历"
```

### 效果

程序对每个"沟通中"的联系人：
- 滚动联系人列表，点击目标联系人
- 验证右侧聊天面板是否切换成功
- 查找"附件简历"（优先）或"在线简历"按钮
- 根据弹窗类型处理：

| 场景 | 检测类型 | 程序行为 |
|------|---------|---------|
| 对方已上传附件简历，可直接下载 | `pdf_preview` | 点击下载 → CDP 拦截 → 文件保存到 `data/resumes/` |
| 需要向对方请求简历，弹窗有"确认"按钮 | `request_popup` | 点击"确认"→ 发出简历请求 → 等对方同意 |
| 简历请求已发出，等待中 | `request_pending` | 跳过，记录"请求中" |
| 沟通不足，无法请求 | `need_reply` | 跳过，记录"需要更多沟通" |
| 对方只有在线简历，弹窗上有附件入口 | `online_resume` | 扫描弹窗上的"附件简历"按钮 → 点击切换 → 尝试下载 |

### 结束后你得到什么

- `data/resumes/` 目录下多出下载的 PDF/Word 简历文件
- 数据库 `resume_operations` 表记录每次操作（downloaded / requested / viewed）
- `candidates` 表更新 `resume_path` 和 `status=resume_downloaded`

---

## 第三步：F7 批量回复未读消息（维护候选人关系）

### 前提

候选人回复了你的消息，产生了**未读消息**。你需要及时回复，维持沟通热度。

### 做什么

在聊天页筛选"未读"消息，提取对话历史，调用 DeepSeek AI 生成上下文相关的自然回复，自动发送。

### 操作

```
① 确保 job_info/company_profile.txt 已写好公司背景
② 输入回复人数（默认 15）
③ 点击 "💬 批量回复未读"
```

### 效果

程序对每个有未读消息的联系人：
- 点击联系人，提取完整聊天消息
- 过滤 UI 噪音（"已读""在线简历"等系统文本）
- 结合公司岗位背景构建 LLM 上下文
- 调用 DeepSeek API 生成回复（≤80 字，禁止索要微信/电话/转账）
- AI 失败时降级到预设模板回复
- 按 Enter 发送（模拟真人操作）
- 写入数据库 `conversations` 表

### AI 回复示例

```
候选人：你好，请问这个岗位还招人吗？
AI回复：您好！岗位还在招聘中。您有金融科技相关的开发经验吗？
---
候选人：薪资大概多少？
AI回复：薪资根据能力和经验面议，包含基本薪资+股权激励。方便了解一下您之前做过的项目吗？
```

### 结束后你得到什么

- 聊天页的未读消息被清空（全部已回复）
- `conversations` 表记录每次对话（候选人消息 + AI 回复）
- `contact_records` 表记录 `action=replied`

## 完整流程总结

```
                     F5                  F6                   F7
                   打招呼              获取简历             批量回复
                     │                   │                    │
  推荐牛人页 ────────►│                   │                    │
  筛选候选人          │                   │                    │
  自动打招呼          │                   │                    │
                     │                   │                    │
  候选人收到招呼 ◄────┘                   │                    │
  部分人回复                             │                    │
  进入"沟通中"状态                        │                    │
                     │                   │                    │
  聊天页 ────────────┼──────────────────►│                    │
  筛选"沟通中"        │                   │                    │
  逐个获取简历        │                   │                    │
                     │                   │                    │
  候选人回复消息 ◄────┼───────────────────┼────────────────────┤
  产生未读消息        │                   │                    │
                     │                   │                    │
  聊天页 ────────────┼───────────────────┼───────────────────►│
  筛选"未读"          │                   │                    │
  AI生成回复          │                   │                    │
  自动发送            │                   │                    │
                     │                   │                    │
  ▼                   ▼                   ▼                    ▼
 candidates 表    resume_operations  conversations 表
 status=contacted  action=downloaded  ai_message
```

### 日常使用节奏

| 时间段 | 操作 | 预期效果 |
|--------|------|---------|
| 每天上午 | F5 打招呼 20-30 人 | 获取新候选人进入沟通池 |
| 每天下午 | F6 获取简历 10 人 | 从已回复的人中拿简历 |
| 每天下午 | F7 回复未读 15 人 | 维护沟通热度，推进面试 |
| 每周 | 查看 Dashboard 统计数据 | 评估转化率，调整筛选策略 |
