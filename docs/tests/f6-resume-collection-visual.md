# F6 简历收集 — 视觉验收清单

> 日期: 2026-06-08
> 功能: 批量获取简历 (POST /api/resume/batch)
> 环境: Docker 容器, VNC 8000, Chrome CDP 9222

## 前置条件

- [ ] Chrome 浏览器已打开并登录 BOSS 直聘
- [ ] 聊天列表中有至少 3 个联系人（含已读和未读）
- [ ] `data/resumes/` 目录存在且可写
- [ ] API 服务运行中 (端口 8002)
- [ ] noVNC 可访问 (端口 8000)

---

## 视觉验证流程

### 步骤 1: API 触发

**操作**: 在驾驶舱或 API 客户端触发 `POST /api/resume/batch`

**截图检查点 #1 — API 响应**:
- [ ] HTTP 200
- [ ] 响应包含 `status`, `downloaded`, `skipped`, `failed`, `total_scanned`
- [ ] 日志显示 `[F6] CDP下载拦截: ok → data/resumes/`

### 步骤 2: 聊天页加载

**操作**: 观察 noVNC 浏览器窗口

**截图检查点 #2 — 聊天页**:
- [ ] URL 为 `zhipin.com/web/chat/*`
- [ ] 左侧联系人列表可见
- [ ] 联系人项目宽度 > 120px, 高度 > 40px
- [ ] 有未读标记的联系人有红点或 "●" 标识

**验证要素**:
| 元素 | 选择器 | 预期状态 |
|------|--------|---------|
| 左侧面板 | `[class*="chat-item"]` (x < 450) | 可见，至少 1 项 |
| 未读标记 | `[class*="unread"], [class*="badge"]` 或 `●` | 至少 1 个 |
| 导航栏 | `a[href*="/web/chat"]` | 高亮/激活状态 |

### 步骤 3: 联系人点击

**操作**: 观察第一个（有未读的）联系人被点击

**截图检查点 #3 — 右侧聊天面板**:
- [ ] 右侧面板显示对应联系人的聊天记录
- [ ] 面板标题包含联系人姓名
- [ ] 消息气泡可见（左侧=对方，右侧=自己）

**验证要素**:
| 元素 | 预期 |
|------|------|
| 聊天面板 | `[class*="chat-detail"]` 内容长度 > 20 |
| 联系人姓名 | 出现在右侧面板 innerText 中 |
| 消息列表 | 至少 1 条消息气泡 |

### 步骤 4: 简历按钮检测

**操作**: 观察 JS 脚本查找简历按钮

**截图检查点 #4 — 简历按钮**:
- [ ] 按钮文本为 "在线简历" / "附件简历" / "查看简历" / "查看附件" 之一
- [ ] 按钮在右侧聊天面板内
- [ ] 按钮可见 (width > 0, height > 0)

**日志验证**:
```
[F6] {name} 简历按钮: {n} 个
```

### 步骤 5: Case 1 — PDF 预览 + 下载

**截图检查点 #5 — PDF 预览**:
- [ ] PDF 内容在页面中渲染（embed/iframe 元素出现）
- [ ] 或新标签页打开

**截图检查点 #6 — 下载确认**:
- [ ] CDP `Browser.downloadProgress` 事件 state="completed"
- [ ] 或 `data/resumes/` 目录中出现新文件
- [ ] 日志显示 `[F6] {name} 简历下载成功 ({size} bytes, method=cdp_event)`

**验证要素**:
| 检查项 | 方法 |
|--------|------|
| PDF 元素 | `document.querySelector('embed[type="application/pdf"]')` |
| 下载按钮 | `[class*="download"], a[download]` |
| 文件存在 | `ls -la data/resumes/` |

### 步骤 6: Case 2 — 请求弹窗

**截图检查点 #7 — 请求弹窗**:
- [ ] 弹窗文本包含 "请求简历" 或 "向牛人"
- [ ] "确认" / "确定" 按钮可见

**截图检查点 #8 — 确认后**:
- [ ] 弹窗关闭
- [ ] PDF 可能弹出（再进入 Case 1 流程）
- [ ] 或显示 "请求已发送"

### 步骤 7: Case 3 — 请求中

**截图检查点 #9 — 请求中状态**:
- [ ] 显示 "请求中" 或 "简历请求" 文本
- [ ] 无确认按钮
- [ ] 日志输出 `[{name}]: 简历请求中，跳过`

### 步骤 8: Case 4 — 沟通不足

**截图检查点 #10 — 沟通不足**:
- [ ] 显示 "双方回复后可以向TA请求" 文本
- [ ] 日志输出 `[{name}]: 沟通不足，无法索取简历`

### 步骤 9: 返回导航

**操作**: 观察 Escape 关闭预览并返回聊天列表

**验证**:
- [ ] 简历预览/弹窗已关闭
- [ ] 聊天列表重新可见
- [ ] 下一个联系人被选中

### 步骤 10: 完成

**截图检查点 #11 — 完成**:
- [ ] `/tmp/f6_final.png` 存在
- [ ] API 返回汇总: `{downloaded, skipped, failed, total_scanned}`
- [ ] DB `resume_operations` 表有对应记录

## 已知限制

1. **无真实 BOSS 账号时**: Case 3 和 Case 4 的视觉验证需要在真实 BOSS 环境中进行
2. **下载拦截降级**: 当 CDP `setDownloadBehavior` 失败时，文件落入 Chrome 默认目录（不在 `data/resumes/`），目录 diff 检测不到
3. **时间依赖**: 4 个固定 sleep (2s/3s/4s) 在网络慢时可能不足，需观察日志确认
4. **联系人坐标**: 每处理一人后 DOM 重渲染，`_refind_contact` 可能失败需回退到原始坐标

## 视觉验收通过标准

- [ ] 11 个截图检查点中至少 8 个通过（Case 3/4 需要真实 BOSS 账号）
- [ ] API 返回 `status: "completed"`
- [ ] 无未捕获的异常日志
- [ ] CDP 连接在处理过程中未断开
- [ ] 每处理 3 个人的进度截图存在 (`/tmp/f6_progress_*.png`)
