## F1: Dashboard 控制面板加载

**用户操作**: 打开浏览器访问 `http://localhost:8321/index.html`

**预期行为**:
1. 页面加载，显示完整的 Dashboard 布局
2. 左侧区域显示 noVNC Web 客户端（嵌入 iframe 或 canvas）
3. 右侧区域显示控制面板，包含所有操作按钮
4. 连接状态显示为 "未连接"（红色圆点）
5. UI 与原型图设计一致：
   - 标题栏: "BOSS直聘自动化管理系统"
   - 状态指示器: 连接状态（红/绿）+ 运行状态文字
   - 功能按钮区: 按顺序排列的功能按钮
   - 日志/输出区: 底部操作日志滚动显示

**视觉验收标准**:
- 页面加载无 4xx/5xx 错误
- 左侧存在 noVNC iframe 或 canvas 元素
- 右侧存在至少 5 个功能按钮
- 连接状态显示 "未连接" 文本
- 页面标题包含 "BOSS直聘"
- **视觉截图**: 与原型 layout 一致（左侧大面板 + 右侧窄面板）

---

## F2: 连接 VNC 远程桌面

**用户操作**: 点击右侧控制面板中的 **[连接VNC桌面]** 按钮

**预期行为**:
1. 前端发起 VNC 连接请求
2. noVNC 客户端连接 Docker 容器的 VNC Server（:1, port 5901）
3. 连接状态: "未连接" → "连接中..." → "已连接"（绿色圆点）
4. VNC 区域内显示 XFCE 桌面画面（壁纸、任务栏可见）

**视觉验收标准**:
- 点击按钮后状态文字变成 "已连接"
- 状态指示灯从红色变为绿色
- noVNC canvas 渲染出画面（非全黑/全白）
- **视觉截图**: noVNC 区域可见桌面元素（任务栏、桌面图标、壁纸）
- 无 WebSocket 连接错误

---

## F3: 打开浏览器进入 Boss直聘网页

**用户操作**: 点击右侧控制面板中的 **[打开Boss直聘]** 按钮

**预期行为**:
1. 后端通过 Chromium 在 Docker 桌面内启动浏览器
2. 浏览器自动导航到 `https://www.zhipin.com`
3. VNC 画面中出现 Chromium 浏览器窗口，显示 Boss直聘首页

**视觉验收标准**:
- API 返回成功响应（200）
- **视觉截图**: VNC 中浏览器窗口可见
- **视觉截图**: 浏览器标题栏或地址栏可见 "zhipin.com" 字样
- 5 秒内浏览器窗口出现在 VNC 画面中
- 无浏览器崩溃或错误页面

---

## F4: Boss直聘登录

**用户操作**: 点击右侧控制面板中的 **[Boss直聘登录]** 按钮

**预期行为**:
1. 后端自动化脚本操控 Docker 内的 Chromium 浏览器
2. 点击 Boss直聘首页 "登录" 按钮
3. 浏览器跳转到 Boss直聘登录页面
4. 如已登录（cookie 有效），直接进入个人中心

**视觉验收标准**:
- API 返回成功响应
- **视觉截图**: 浏览器显示登录页面（手机号/密码输入框 或 扫码区域）
- **视觉截图**: 页面含 "登录" / "密码登录" 文本 或 URL 含 "login"
- 如已登录：页面显示求职者列表/搜索界面
- 无弹窗错误

---

## F5: 批量主动打招呼

**用户操作**: 点击右侧控制面板中的 **[批量主动打招呼]** 按钮

**预期行为**:
1. 打招呼自动化脚本启动
2. 浏览器在 Boss直聘 搜索/推荐页按筛选条件筛选候选人
3. 对符合条件者逐一点击 "打招呼"，发送预设招呼语
4. VNC 中可见浏览器持续操作：滚动列表 → 点击按钮 → 发送消息
5. 底部日志区实时输出操作日志

**视觉验收标准**:
- API 返回启动成功
- **视觉截图**: VNC 中可见 Boss直聘搜索列表（候选人卡片）
- **连续截图**: 每 5 秒一张，可见页面状态变化（不同候选人被操作）
- 日志区有打招呼输出
- 完成后有统计（成功/跳过/失败数量）
- 不出现浏览器崩溃或页面卡死

---

## F6: 批量获取简历 & 批量聊天回复

### F6a: 批量获取简历

**用户操作**: 点击 **[批量获取简历]** 按钮

**预期行为**:
1. 脚本识别 "已发简历" 和 "未发简历" 候选人
2. 对未发简历者逐一点击 "获取简历"
3. 下载简历到本地 `data/resumes/` 目录

**视觉验收标准**:
- API 返回启动成功
- **视觉截图**: VNC 中可见候选人列表/详情页
- `data/resumes/` 有新下载文件
- 日志区有下载统计

### F6b: 批量聊天回复

**用户操作**: 点击 **[批量聊天回复]** 按钮

**预期行为**:
1. 脚本识别未读消息
2. 按话术模板库智能匹配回复
3. 逐条回复未读消息

**视觉验收标准**:
- API 返回启动成功
- **视觉截图**: VNC 中可见聊天消息页面
- **视觉截图**: 可见已发送的回复消息气泡
- 日志区有回复统计
- 话术匹配合理
