## 一、问题分析

### 1.1 原始问题
- **OCR识别有空格**：`"中 山大 学"` → 应匹配 `"中山大学"`
- **误匹配普通学校**：`"广州职业"`、`"重庆人文"` 等被错误点击
- **短名称误匹配**：`"中南大学"` 被误匹配为 `"南大"`

### 1.2 根本原因
1. 旧版只做简单的去空格处理，未考虑其他特殊字符
2. 缺少黑名单机制，无法排除普通学校关键词
3. 短名称（≤3字符）匹配逻辑不够严格，容易误匹配

---

## 二、优化方案

### 2.1 核心改进

#### ① 增强文本清理
```python
# 旧版：只去除空格
text_no_space = text.replace(" ", "").replace("　", "")

# 优化版：去除所有空格、标点、特殊字符
text_clean = re.sub(r'[\s\u3000\-_·•]', '', text)
text_clean = text_clean.lower()  # 英文转小写
```

#### ② 新增黑名单机制
```python
SCHOOL_BLACKLIST_KEYWORDS = [
    "职业", "职业技术", "专科", "高职", "技师", "技工",
    "人文", "科技学院", "学院", "民办", "独立学院",
    "电大", "函授", "成人", "自考", "网络教育"
]

# 黑名单检查：如果包含普通学校关键词，直接拒绝
for keyword in SCHOOL_BLACKLIST_KEYWORDS:
    if keyword in text_clean:
        return False
```

#### ③ 严格的短名称匹配
```python
# 短名称（≤3个字符）需要更严格的验证
if len(school_clean) <= 3:
    idx = text_clean.find(school_clean)
    before = text_clean[idx-1] if idx > 0 else ' '
    
    # 检查前面是否有汉字（可能是另一个学校的一部分）
    # 例如："中南大学"包含"南大"，但"南大"前面有"中"，应该跳过
    if before and '\u4e00' <= before <= '\u9fff':  # 前面是汉字
        continue  # 跳过，可能是误匹配
```

### 2.2 匹配优先级

1. **完全匹配**（最高优先级）：`school_clean == text_clean`
2. **长名称匹配**（>3字符）：直接包含即可
3. **短名称匹配**（≤3字符）：需要验证前后字符，防止误匹配
