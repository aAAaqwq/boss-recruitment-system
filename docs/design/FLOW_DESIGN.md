# BOSS直聘 AI 招聘全流程设计 v1.0
## 从推荐牛人到获取简历的完整流程

---

## 一、核心流程概述

```
推荐牛人 → 打招呼(自动) → 候选人回复 → 进入沟通 → 查看简历 → 获取简历(下载/请求) → 换微信 → 下一人
```

BOSS直聘的**完整招聘流程**不是一进入就点"沟通"那么简单。需要先理解平台的逻辑：

---

## 二、每个步骤的详细拆解

### Step 0: 登录态确认
- **前提**：Chrome已登录BOSS直聘（cookie有效）
- **验证**：在 `https://www.zhipin.com/` 可以看到招聘后台

### Step 1: 进入"推荐牛人"页面
- **URL**: `https://www.zhipin.com/web/chat/geek`
- **动作**：左侧导航栏点击 **"推荐牛人"** （第二图标，x≈45, y≈295）
- **页签**：默认"推荐"页签（不要切到"最新"）
- **关键区域**：
  - 候选人卡片列表（多个候选人，每个卡片有"立即打招呼"按钮）
  - 每个卡片显示：姓名、年龄、学校/公司、岗位、标签
  
### Step 2: 打招呼（批量）
- **动作**：对每个候选人卡片点击 **"立即打招呼"** 按钮（蓝色、湖蓝色）
- **作用**：系统自动发送预设打招呼消息
- **效果**：候选人→进入"沟通"列表（右侧侧边栏能看到新消息）
- **注意**：BOSS直聘每个候选人都需要先打招呼，才能进入沟通对话

### Step 3: 进入"沟通"页面
- **URL**: `https://www.zhipin.com/web/chat/index`
- **入口**：左侧导航栏点击 **"沟通"**（第四图标，x≈45, y≈370）
- **左侧列表**：显示所有已打招呼/聊天的候选人
  - 第一个条目 = 最近联系的候选人
  - 每项显示：头像、姓名、职位、最后消息、时间
- **右侧详情区**：
  - 顶部：候选人概要信息（姓名、年龄、学校、标签）
  - 右上角："在线简历"和"附件简历"按钮
  - 中间：聊天对话框
  - 底部：快捷回复 + 输入框 + 发送按钮

### Step 4: 点击候选人（列表遍历）
- **动作**：从左到右点击左侧候选人列表中的候选人
- **要点**：
  - 第一个候选人坐标：x≈330, y≈355（第二列列表）
  - 每个候选人间距≈68px
  - 点击后右侧刷新显示该候选人的聊天详情

### Step 5: 判断简历状态（核心！）
- **按钮**：右上角的"附件简历"按钮
  - **深蓝色** → 候选人已发送附件简历 → 点击后可预览和下载
  - **浅蓝色/灰色** → 候选人未发送 → 点击后会弹出"确认发送请求"弹窗

### Step 6a: 深蓝按钮 → 下载简历
- **点击"附件简历"**（深蓝色）
- **弹窗**：显示PDF简历预览（白色背景大框）
- **点击"下载"**（右上角，x≈1550, y≈135）
- **下载完成**后：
  - 点击**预览外部灰色区域**（x≈1300, y≈540）
  - 按ESC退出预览

### Step 6b: 浅蓝按钮 → 请求简历
- **点击"附件简历"**（浅蓝色/灰色）
- **弹窗**：确认发送请求（"是否向TA发送附件简历请求？"）
- **点击"确认"**（绿色按钮，x≈1175, y≈815）
- 系统自动发送请求消息

### Step 7: 换微信
- **点击"换微信"**（x≈630, y≈770）
- **弹窗**：发送微信交换请求
- **点击"确认"**（绿色按钮）
- 或者Boss直聘的自动微信交换流程

### Step 8: 轮转到下一候选人
- **鼠标滚轮**向下滚动（或点击左侧列表的下一个人）
- 重复 Step 4-7

## 六、UI状态检测要点

本系统依赖的检测逻辑：

1. **导航栏状态**：点"推荐牛人"后，该图标变为绿色背景高亮
2. **按钮颜色**：
   - 深蓝(附件简历) → B > 150 && B > R && B > G
   - 浅蓝(附件简历) → G > 150 && B > G && R > 120
   - 灰色 → R≈G≈B 且 < 150
3. **弹窗检测**：
   - 简历预览：白色区域亮度 > 200
   - 确认弹窗：确认按钮绿色 (G > 180 && G > R)
4. **换微信按钮**：浅绿色，点击后弹确认窗
