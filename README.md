# BOSS 直聘招聘自动化系统

## 环境构建

### 1. 前置条件

- Docker + Docker Compose
- DeepSeek API Key

### 2. 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 填入 DeepSeek API Key：

```env
DEEPSEEK_API_KEY=sk-xxxxxxxx
PG_HOST=postgres
PG_PORT=5432
PG_DB=boss_recruitment
PG_USER=boss
PG_PASSWORD=boss123
API_SECRET_KEY=your-random-key-at-least-32-chars
API_USERNAME=admin
API_PASSWORD=boss-recruit-2026
```

### 3. 构建并启动

```bash
docker-compose up -d --build
```

### 4. 访问

| 地址 | 用途 |
|------|------|
| http://localhost:3101 | 管理界面（驾驶舱） |
| http://localhost:8002 | API 服务 |
| http://localhost:6901 | noVNC 远程桌面 |

### 5. 登录

- 用户名：`admin`
- 密码：`.env` 中 `API_PASSWORD` 的值，默认 `boss-recruit-2026`

## 前端界面说明

### 左侧区域 — 浏览器控制

| 按钮 | 功能 |
|------|------|
| 🚀 打开BOSS直聘 | 在 VNC 浏览器中打开 zhipin.com，需先用 noVNC 手动扫码登录 |
| 🔗 连接VNC | 连接本地 Chrome CDP 调试端口 |
| 🔍 检查登录 | 检测当前 BOSS 直聘页面登录状态 |
| 📤 导出Cookies | 导出当前浏览器 Cookies 到文件 |
| 📥 导入Cookies | 从文件导入 Cookies 到浏览器 |

### 左侧区域 — 工作流

| 按钮 | 功能 | 说明 |
|------|------|------|
| 👋 筛选+打招呼 | F5 — 按筛选条件扫描推荐页候选人并自动打招呼 | 需先配置筛选条件 |
| 📄 批量获取简历 | F6 — 批量向沟通中的候选人请求/下载简历 | 需要 BOSS 账号有开料权限 |
| 💬 批量回复未读 | F7 — 对未读消息批量 AI 回复 | 需要 DeepSeek API 可用 |
| 📥 下载已获取简历 | F8 — 下载已同意分享简历的联系人的附件简历 | 针对已获取简历的联系人 |

**每个按钮旁有数字输入框，控制处理人数上限。**

### 右侧区域 — 配置面板

**筛选条件：**

| 配置项 | 说明 |
|--------|------|
| 学历 | 最低学历要求（博士/硕士/本科/大专） |
| 工作年限 | 最低工作年限（1/2/3/5/10 年） |
| 学校白名单 | 国内 33 所 + 海外 99 所名校，支持多选 |
| 包含关键词 | 职位/技能关键字，匹配的才联系 |
| 排除关键词 | 排除关键字，命中的跳过 |

**话术模板：**

| 配置项 | 说明 |
|--------|------|
| 打招呼话术 | 筛选+打招呼 F5 使用的首次联系话术 |
| 求简历话术 | 批量获取简历 F6 使用的请求话术 |
| 跟进话术 | 批量回复 F7 使用的跟进话术 |

**岗位模板：** 用于 AI 生成回复时的岗位背景信息，可切换不同岗位文件。

### 顶部状态栏

- 显示招呼数、简历数、已回复数实时统计
- 任务执行状态和进度
- ⛔ 取消任务按钮（任务运行中时可用）

### 底部 — VNC 远程桌面

- 🔄 刷新 / 🔍± 缩放 / 📷 截图
- 用于手动操作 BOSS 直聘页面（扫码登录、手动调整等）

### 底部 — 运行日志

- 实时显示所有工作流运行日志
- 🗑️ 清除 / ▲▼ 展开收起

## 使用流程

### 首次使用

1. 打开 `http://localhost:3101`，登录
2. 点击连接到桌面，然后选VNC.html点击 "🚀 打开BOSS直聘"
3. 在 BOSS 页面手动扫码登录
4. 配置筛选条件（学历、年限、学校白名单等）
5. 配置话术模板

### 日常使用

1. 点击 "👋 筛选+打招呼"，输入人数，确认
2. 等待完成后，点击 "📄 批量获取简历"
3. 有未读消息时，点击 "💬 批量回复未读"
4. 有已获取简历时，点击 "📥 下载已获取简历"

## 常见错误

### 浏览器未连接

**现象：** 工作流启动后提示 "浏览器未连接"

**原因：** Chrome CDP 调试端口（9222）未开启或 BOSS 直聘页面未打开

**解决：** 先用 noVNC 打开 BOSS 直聘并登录，或点击 "🚀 打开BOSS直聘"

### 每日上限

**现象：** 打招呼任务显示 "今日已达上限"

**原因：** 达到了 `.env` 中 `DAILY_CONTACT_CAP` 设置的每日联系人数上限

**解决：** 等待次日，或调大 `DAILY_CONTACT_CAP` 值

### 限制弹窗

**现象：** 任务中途停止，日志提示 "检测到限制弹窗"

**原因：** BOSS 平台弹出了限制提示（次数用尽、权益不足、需要会员等）

**解决：** 手动处理弹窗后重新运行，或等待次日恢复

### 未获取到简历

**现象：** F6 结果为 0 或大量跳过，日志显示 "无简历按钮" 或 "沟通不足"

**原因：** 候选人未回复或沟通轮次不够，不支持请求简历

**解决：** 先用 F7 批量回复，沟通轮次够了再跑 F6

### 下载超时

**现象：** 点击下载后超时，未检测到文件

**原因：** 浏览器下载路径未正确设置或 PDF 预览加载慢

**解决：** 重试一次，系统已优化下载检测逻辑

### DeepSeek API 不可用

**现象：** F7 回复结果全是失败，日志提示 API 错误

**原因：** API Key 未配置或余额不足

**解决：** 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确，确认账户余额

## 数据库

- PostgreSQL 15，端口 5432
- 数据库名：`boss_recruitment`，用户：`boss`
- 命令行访问：`docker exec -it boss-postgres psql -U boss -d boss_recruitment`
- GUI 客户端（DBeaver/pgAdmin）：连接 `localhost:5432`，同上用户名密码

### 表结构

| 表 | 用途 |
|----|------|
| candidates | 候选人库 |
| contact_records | 联系记录 |
| chat_sessions | 对话会话 |
| resume_operations | 简历操作记录 |
| conversations | AI 对话记录 |
| runtime_state | 运行时配置 |
| users | 系统登录用户 |
| boss_accounts | BOSS 多账号配置（预留） |
| reply_templates | 回复话术模板 |
| processed_candidates | 已处理去重 |
